This guide will help you deploy the Image Generator to GitHub Pages.

Step 1: Prepare Your Files

Make sure you have these files in your repository:

  • index.html - Main application file
  • static/ - CSS, JavaScript, and images
  • reference_images/ - Reference images (optional)

Step 2: Create/Update .gitignore

Add these lines to exclude unnecessary files:

# Python backend (not needed for static version)
*.pyc
__pycache__/
venv/
.env

# Keep only static files
src/
templates/
config/
requirements.txt

Step 3: Commit Your Changes

git add index.html static/ reference_images/
git commit -m "Deploy static image generator to GitHub Pages"
git push origin main

Step 4: Enable GitHub Pages

  1. Go to your repository on GitHub
  2. Click Settings
  3. Scroll to Pages section
  4. Under "Source", select main branch
  5. Click Save

üîë Getting an OpenRouter API Key

For You (Developer)

  1. Visit OpenRouter.ai
  2. Sign up or log in
  3. Go to Keys section
  4. Create a new API key
  5. Add credits to your account

For Your Users

Users will need to:

  1. Get their own OpenRouter API key (same process as above)
  2. Click "API Key" in the sidebar of your deployed app
  3. Enter their API key
  4. Start generating images

  1. Visit your GitHub Pages URL

  4. Check browser console for any errors

üîß Troubleshooting

Common Issues

404 Error:

  • Wait a few minutes for GitHub Pages to build
  • Check that index.html is in the root directory
  • Verify GitHub Pages is enabled in settings

Blank Page:

  • Check browser console for errors (F12)
  • Verify all file paths are correct (case-sensitive)
  • Ensure JavaScript modules are loading

Images Not Generating:

  • Verify API key is correct
  • Check OpenRouter account has credits
  • Look for CORS errors in console
